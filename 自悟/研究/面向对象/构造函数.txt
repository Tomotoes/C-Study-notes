你能信，我对构造函数 陌生！
先说 修饰符，可以用其他类型，private（不能实例对象）但最好 用public

构造函数格式：修饰符 类名 ( ) { }

当然 最重要的是 构造函数的执行顺序 会先执行 父类的构造函数，再执行子类的构造函数

NO！最重要的来了，构造函数 可以像 类一样继承！

一般来说 都是隐式继承，也刚好解释了 上面 构造函数的调用顺序。

下面来点硬菜：构造函数的显式继承

class Pet
{
  public Pet（string name）
	{
		Name=name;
	}
}

class Dog : Pet
    {

        public Dog(string name):base(name)
        {

        }
     }

等到下次 实例化子类的对象时，就会 自动调用父类的构造函数。
